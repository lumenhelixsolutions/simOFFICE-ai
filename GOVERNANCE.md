# Governance

SimOffice is maintained as an open-source project with a commercial-grade product direction.

## Maintainer principles

- Keep the project usable by independent builders.
- Keep the core open.
- Do not hide critical runtime behavior behind vague UI states.
- Prefer explicit human approval for irreversible actions.
- Maintain a clear path from prototype to production.

## Decision model

Small fixes can be merged quickly. Larger changes should include an issue or design note explaining:

- User value.
- Runtime impact.
- Security impact.
- UI impact.
- Backward compatibility.
